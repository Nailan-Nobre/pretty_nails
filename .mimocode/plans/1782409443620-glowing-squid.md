# Plano: Notificações Push Nativas (Android/iOS) - Pretty Nails

## Diagnóstico: Por que não funciona hoje

Encontrei **3 causas raiz** que impedem qualquer notificação push de chegar:

### Causa #1 (CRÍTICA): `ONESIGNAL_APP_ID` não é passado no build
**Arquivo:** `front_end/build.sh:12`

```bash
flutter build web --dart-define=BACKEND_URL=$BACKEND_URL
```

O build **só passa `BACKEND_URL`**. O `ONESIGNAL_APP_ID` NÃO é incluído no `--dart-define`. Como `onesignal_service.dart:6-8` faz:

```dart
static const String _appId = String.fromEnvironment('ONESIGNAL_APP_ID', defaultValue: '');
static Future<void> init() async {
  if (_appId.isEmpty) return;  // ← AQUI: retorna sem fazer nada
```

**Resultado:** OneSignal nunca é inicializado. Zero notificações recebidas.

### Causa #2 (BUG): Backend usa `notificacoes_email` para bloquear push
**Arquivo:** `back_end/src/services/onesignalService.js:86-89`

```js
if (manicure.notificacoes_email === false) {
    console.log('Manicure desativou notificações');
    return;  // ← BLOQUEIA push também!
}
```

A configuração de notificações no app (`settings_screen.dart:51`) faz:
```dart
await ApiService.put('/auth/profile', body: {
  'notificacoes_email': _notifEnabled && type == 'email',  // false quando tipo é 'app'
});
```

Quando a manicure escolhe "Notificações no App" (push), `notificacoes_email` fica `false`. O backend interpreta isso como "desativou TUDO" e **bloqueia push também**.

### Causa #3: iOS sem configuração de push
**Arquivo:** `front_end/ios/Runner/Info.plist` — Falta `UIBackgroundModes` com `remote-notification`. Sem isso, iOS não recebe push em background/kill.

### Fase 2: Corrigir Bug do Backend (Push vs Email)

**Objetivo:** Separar a lógica de push e email para que funcionem independentemente.

**2.1 Corrigir `back_end/src/services/onesignalService.js`**
- Remover a verificação de `notificacoes_email` do código de push
- Push e email são canais independentes
- A manicure pode ter push ATIVADO e email DESATIVADO (ou vice-versa)

Mudar de:
```js
if (manicure.notificacoes_email === false) {
    console.log('Manicure desativou notificações');
    return;
}
```

Para:
```js
// Push notifications são independentes do email
// A preferência de push é controlada pelo app (SharedPreferences)
```

**Arquivos modificados:**
- `back_end/src/services/onesignalService.js`

### Fase 3: Configurar Build Nativo Android (APK)

**Objetivo:** Gerar um APK instalável com OneSignal funcionando.

O app já tem toda a infraestrutura nativa configurada:
- `AndroidManifest.xml` já declara `POST_NOTIFICATIONS` (permissão Android 13+)
- `onesignal_flutter: ^5.2.0` já está no pubspec.yaml
- `OneSignalService` já inicializa e registra o player ID
- `NotificationService` já cria o canal de notificações

**3.1 Comando para gerar APK com OneSignal:**
```bash
cd front_end
flutter build apk --dart-define=BACKEND_URL=https://pretty-nails-do11.vercel.app --dart-define=ONESIGNAL_APP_ID=c165041f-74ec-4790-b66e-aaaa78601453
```

O APK gerado em `build/app/outputs/flutter-apk/app-release.apk` pode ser instalado diretamente no Android.

**3.2 Configuração OneSignal Dashboard (Android)**
- Verificar no painel do OneSignal se o app Android está registrado
- Se não estiver, configurar com Firebase Cloud Messaging (FCM):
  1. Criar projeto no Firebase Console
  2. Baixar `google-services.json`
  3. Colocar em `front_end/android/app/`
  4. Adicionar credenciais FCM no OneSignal Dashboard

**Arquivos não modificados** (já estão corretos):
- `front_end/android/app/src/main/AndroidManifest.xml`
- `front_end/android/app/build.gradle.kts`
- `front_end/lib/services/onesignal_service.dart`

### Fase 4: Configurar Build Nativo iOS (futuro — requer Apple Developer)

**Objetivo:** Preparar tudo para quando a conta Apple Developer estiver disponível.

**Pré-requisito:** Conta Apple Developer Program (R$ 99/ano)

**4.1 Adicionar `UIBackgroundModes` ao Info.plist**

- Adicionar chave `UIBackgroundModes` com valor `['remote-notification']`

1. No Apple Developer Portal → Certificates → Create APNS Key
2. Baixar a `.p8` key
3. No OneSignal Dashboard → Settings → Platforms → Apple iOS

1. Adicionar app iOS no projeto Firebase existente
2. Baixar `GoogleService-Info.plist`
3. Colocar em `front_end/ios/Runner/`
4. Adicionar `firebase_core` ao pubspec.yaml
5. Configurar Firebase no `main.dart` antes do OneSignal

**4.4 Comando para gerar IPA:**
```bash
flutter build ios --dart-define=BACKEND_URL=... --dart-define=ONESIGNAL_APP_ID=...
```

**Arquivos a modificar:**

## Nota sobre Flutter Web vs Nativo

O app atualmente é deployado como **Flutter Web** no Vercel. Os plugins `onesignal_flutter` e `flutter_local_notifications` **só funcionam em builds nativos** (Android/iOS). No web, eles são silenciosamente ignorados.

Para ter notificações push, é necessário gerar APK (Android) ou IPA (iOS) e distribuir aos usuários. O build web continua funcionando para o link público de agendamento dos clientes.
