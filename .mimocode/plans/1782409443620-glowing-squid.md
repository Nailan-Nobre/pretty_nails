# Plano: Corrigir permissão, sincronização e verificação OneSignal

## 4 Problemas identificados

### 1. Prompt de permissão usa comportamento forçado
`requestPermission(true)` força o diálogo imediatamente. O padrão (`requestPermission()`) é mais natural.

### 2. Permissão aparece antes do login ser confirmado
Tanto `main.dart` (startup) quanto `auth_service.dart` (logo após login API) chamam `requestPermission()`. Deveria só acontecer quando o usuário vê a home screen.

### 3. Toggle das configurações não sincroniza com permissão do SO
Ligar/desligar no settings só muda SharedPreferences. Não controla OneSignal subscription nem pede permissão do SO.

### 4. Verificação no OneSignal dashboard
Backend envia push mas usuário quer confirmar que chegou. OneSignal dashboard > Messages mostra mensagens enviadas.

---

## Alterações

### `onesignal_service.dart`
- Trocar `requestPermission(true)` → `requestPermission()` (padrão, sem forçar)
- Adicionar `isPermissionGranted()` para checar estado atual

### `main.dart`
- Remover `requestPermission()` do startup
- Manter `sendPlayerIdToServer()` se logado
- Pedir permissão só quando home carrega

### `auth_service.dart`
- Remover `requestPermission()` e `sendPlayerIdToServer()` do login
- Essas ações acontecem na home screen

### `settings_screen.dart`
- Ao ligar "Notificações no App": solicitar permissão do SO + optIn OneSignal
- Ao desligar: optOut OneSignal
- Ao trocar tipo: sincronizar corretamente

### Backend (diagnóstico)
- Adicionar log com player_id antes de enviar push

## Verificação
- Flutter run → login → home aparece → permissão solicitada naturalmente
- Configurações → toggle ON/OFF → OneSignal dashboard reflete mudanças
- Criar agendamento → backend loga player_id e resultado do envio
